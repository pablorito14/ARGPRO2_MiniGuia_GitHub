Mini-Guia GitHub

1) Entrar en GitHub y crear un repositorio nuevo. Poner un nombre (y descripción si se quiere), dejar en publico (para poder desplegar en Pages) y crear

2) Vamos a ver los comandos que vamos a tener que ejecutar en la consola.

3) En vs code dar click derecho sobre la carpeta de nuestro proyecto y elegir Abrir en terminal integrado y esperar a que en la consola salga la carpeta que queremos subir. Prestar atención que este dentro de esa carpeta, sino va a haber problemas.

4) Se ejecuta cada linea del punto 2 por separado. La primera linea es simplemente un mensaje y no es necesario ejecutar. Prestar atención a que la 3ra linea, en lugar de usar git add README.md tenemos que poner git add . para que tome todos los archivos de nuestro proyecto

Lo que se muestra debajo del git commit va a depender de la cantidad de archivos de nuestro proyecto

5) Volviendo a abrir nuestro repositorio en GitHub (o actualizando si quedo ahi) se van a ver los archivos de nuestro proyecto. Es fundamental que el index.html este en la raíz (como se ven en la imagen para desplegar correctamente en GitHub Pages)

6) Para desplegar en GitHub Pages hay que ir a Settings, Pages y en la derecha elegir main y / (root). Guardar y esperar un par de minutos para que se despliegue.

Después de un par de minutos actualizar la página y en la parte superior se va a ver el link con el botón para ir a la web que se subió.

INFO EXTRA 1: A medida que se modifican archivos se van a mostrar en color verde los nuevos y en amarillo/naranja los modificados desde el ultimo commit

INFO EXTRA 2: Para que funcionen correctamente los recursos (css, js e imagenes) es necesario usar ./ al comienzo de la ruta. Por ejemplo:

styles.css en la misma carpeta que el index.html => src="./styles.css"
styles.css dentro de una carpeta llamada css => src="./css/styles.css"
